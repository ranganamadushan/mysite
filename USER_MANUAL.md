# 🚀 Academic Portfolio Automation Engine

Welcome to your fully-automated, professional 6-page academic portfolio website! 

This website uses a completely static HTML architecture for lightning-fast loading speeds, but is entirely managed by a robust local **Python Build Engine** (`update_portfolio.py`). 

Instead of manually editing HTML code every time you get a new job or publish a paper, simply feed your data to the engine, and it handles everything from API scraping to deploying to GitHub!

---

## 🏗️ Architecture Layout

The website is separated into six elegant, seamlessly linked pages:
1. `index.html` — Hero Landing Page & Dashboard
2. `education.html` — Fully formatted Education History
3. `experience.html` — Professional Career Timeline
4. `projects.html` — Hardware & Custom Research Showcase
5. `applications.html` — Desktop Utilities Hub
6. `publications.html` — Publications List

All of these pages are overwritten and continuously generated by the Python Engine whenever it is run.

---

## 🛠️ How to Edit Your Content

You have three primary ways to update your information across the site depending on what you want to change:

### 1. Education & Experience (via LinkedIn)
To effortlessly update your Timeline or Educational degree history:
1. Go to your **LinkedIn Profile** in your browser.
2. Click the **"More..."** button and select **"Save to PDF"**.
3. Save the downloaded file directly as `Profile.pdf` inside this folder (`g:\My Drive\MYSITE\engine\`), replacing the old one.
4. Run the engine (see deployment instructions below). The script will parse the PDF natively and extract your newest jobs seamlessly!

### 2. Custom Data (`data.json`)
The `data.json` file is your central offline database for custom information. Open it in a text editor like VS Code or Notepad to safely modify it:
*   **Projects**: To add a new research project, copy an existing block inside the `"projects"` array and paste it below.
*   **Applications**: To list a new desktop tool across the site, add it to the `"applications"` array. Include `"download": true` (for .exe) or `"website": true` (for URL linking).
*   **Site Visits**: Need to forcibly bump your site visitor count? Edit the integer next to `"site_visits"`.
*   **Research Interests**: Add or remove text tags in the `"research_interests"` list to customize your landing page dashboard.

### 3. Publications & Metrics (via Google Scholar)
You never need to edit your publications manually! The engine automatically fetches your latest papers, h-index, i10-index, and total citations securely from your Google Scholar profile ID every single time you run the script.

---

## ⚡ How to Push Updates Live

Whenever you edit `data.json` or swap out your `Profile.pdf`, follow these 2 simple steps to safely push the changes to the live internet:

1. Open a terminal or **Command Prompt** inside this folder.
2. Run the automation engine:
   ```cmd
   python update_portfolio.py
   (Note: Always run this explicitly from inside the `engine` folder!)
   ```

### What exactly does the engine do?
1. Parses your PDF and custom JSON data.
2. Connects to the internet to fetch live Google Scholar metric updates.
3. Automatically assembles and formats all the complex HTML pages.
4. Leaves a permanent timestamped trail in the `website_updates.log` audit file.
5. **Automatically Stages, Commits, and safely Auto-Pushes your entire live site code specifically to GitHub!**

*(Note: Live updates on GitHub Pages actively take about 60 to 90 seconds to process. Always perform a Hard Refresh `Ctrl + Shift + R` in your browser to forcefully view the newest version!)*
